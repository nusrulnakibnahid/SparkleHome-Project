## 🏠 About The Project

**SparkleHome** is a fully responsive, multi-section landing page for a professional cleaning service business. Designed with conversion in mind, the site showcases services, team members, pricing packages, customer reviews, and a quote request form — all wrapped in a clean, trustworthy brand identity.

> *"Professional cleaning you can trust."*

Whether you're a freelance developer referencing a real-world landing page structure, or a business owner adapting this template for your brand — SparkleHome is clean, production-ready, and easy to customize.

## 🚀 Getting Started

### Prerequisites

All you need is a modern web browser (Chrome, Firefox, Edge, Safari).

### Installation

**1. Clone the repository**
```bash
git clone https://github.com/nusrulnakibnahid/sparklehome.git
```

**2. Navigate to the project folder**
```bash
cd sparklehome
```

**3. Open in your browser**
```bash
# macOS
open index.html

# Windows
start index.html

# Linux
xdg-open index.html
```

**Or use a local dev server (recommended for best experience):**
```bash
# Option A — VS Code: install the "Live Server" extension, then click "Go Live"

# Option B — Node.js
npx serve .

# Option C — Python
python -m http.server 5500
```

Then open → `http://localhost:5500`

### 🔤 Typography

```css
/* Headings — Bold, clear, high-impact */
font-family: 'Poppins', sans-serif;
font-weight: 700 | 600;

/* Body — Readable, neutral */
font-family: 'Open Sans', sans-serif;
font-weight: 400 | 500;
```

| Element | Font | Size | Weight | Color |
|---|---|---|---|---|
| H1 (Hero) | Poppins | 48–64px | 700 | `#FFFFFF` |
| H2 (Section) | Poppins | 32–40px | 700 | `#222222` |
| H3 (Card title) | Poppins | 20–24px | 600 | `#222222` |
| Body text | Open Sans | 15–16px | 400 | `#666666` |
| Button label | Poppins | 14–16px | 600 | `#FFFFFF` / `#222222` |
| Badge / Tag | Open Sans | 12px | 600 | contextual |

---

### 📐 Spacing System

All spacing follows an **8pt grid** system:

| Token | Value | Usage |
|---|---|---|
| `--space-xs` | `8px` | Inner padding, icon gaps |
| `--space-sm` | `16px` | Button padding, small gaps |
| `--space-md` | `24px` | Card padding, form fields |
| `--space-lg` | `48px` | Section inner padding |
| `--space-xl` | `80px` | Section vertical spacing |
| `--space-xxl` | `120px` | Hero top/bottom padding |

### 🔘 Button Styles

| Variant | Background | Text | Border | Use Case |
|---|---|---|---|---|
| **Primary** | `#1A73E8` | `#FFFFFF` | none | Main CTA, Get Free Quote |
| **Secondary** | `#FFD600` | `#222222` | none | Accent actions, hover states |

### 📱 Breakpoints (Responsive)

| Name | Width | Description |
|---|---|---|
| Mobile | `< 480px` | Single column, stacked layout |
| Tablet | `481px – 768px` | 2-column grids |
| Small Desktop | `769px – 1024px` | 3-column grids |
| Large Desktop | `> 1024px` | Full layout, max-width container |

```css
/* Container */
max-width: 1200px;
margin: 0 auto;
padding: 0 24px;
```

---

### 🖼 Image Guidelines

- **Hero background** — minimum `1920×1080px`, use dark overlay at `rgba(0,0,0,0.5)`
- **Service icons** — `80×80px`, SVG preferred
- **Team photos** — square aspect ratio `300×300px`, circular crop via `border-radius: 50%`
- **Review avatars** — `60×60px`, circular crop
- **File format** — `.webp` preferred for performance; `.jpg` fallback

---

### ✍️ Writing Style

- Use **sentence case** for headings (not Title Case)
- Keep CTAs action-oriented: *"Get Your Free Quote"*, *"Book Now"*, *"Learn More"*
- Use italics for the brand tagline only
- Avoid jargon; write for a general audience

## 🤝 Contributing

Contributions are warmly welcome! To get started:

1. **Fork** the repository
2. **Create** a feature branch
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Commit** your changes with a clear message
   ```bash
   git commit -m "feat: add mobile hamburger menu"
   ```
4. **Push** to your branch
   ```bash
   git push origin feature/your-feature-name
   ```
5. **Open a Pull Request** and describe your changes

### Commit Message Convention

```
feat:     New feature
fix:      Bug fix
style:    CSS/UI changes (no logic change)
docs:     Documentation updates
refactor: Code restructuring
chore:    Maintenance tasks
```
